Update
sudo ./scripts/update-wg.sh

Backs up first, apt upgrade, restarts WireGuard, verifies it restarted cleanly

Cryptography

Key Exchange
Curve25519
256-bit elliptic-curve Diffie-Hellman. Modern, patent-free. Private key is 32 random bytes; public key derived via wg pubkey.
Encryption
ChaCha20-Poly1305
Authenticated encryption. Faster than AES on devices without AES-NI hardware. Provides both confidentiality and integrity.
Hashing
BLAKE2s
Faster than SHA-256. Used in the Noise protocol handshake hash chain and key derivation.
Preshared Key — Each client also gets a 256-bit symmetric preshared key (wg genpsk). This provides an additional layer of post-quantum resistance on top of the Curve25519 handshake.

After 2 messages both sides independently derive identical transport keys (ChaCha20). Keys rotate every 3 minutes or 2⁶⁴ packets via a new handshake. Ephemeral keys provide forward secrecy.

Full Tunnel Packet Flow

Client app → sends packet to 8.8.8.8
↓ AllowedIPs = 0.0.0.0/0 → route via wg0
WireGuard (client) → encrypts with ChaCha20-Poly1305 + counter nonce
↓ UDP to server:51820
WireGuard (server) → decrypts, validates counter & AllowedIPs
↓ iptables NAT MASQUERADE rewrites source 10.100.0.2 → server public IP
Internet → sees request from server's public IP, replies to server
↓ NAT table reverses: server IP → 10.100.0.2 → re-encrypt → send to client

wg0.conf Breakdown

[Interface]
PrivateKey = <server private key>     # Curve25519 — 32 bytes, Base64
Address    = 10.100.0.1/24            # Server VPN IP (also owns the subnet)
Address    = fd00:100::1/64           # IPv6 equivalent
ListenPort = 51820                    # UDP port
MTU        = 1420                     # 1500 - 80 bytes WG overhead

# NAT — applied when interface comes UP
PostUp  = iptables -t nat -A POSTROUTING -s 10.100.0.0/24 -o eth0 -j MASQUERADE
PostUp  = iptables -A FORWARD -i %i -j ACCEPT   # Allow VPN→internet
PostUp  = iptables -A FORWARD -o %i -j ACCEPT   # Allow internet→VPN

# Cleanup — applied when interface goes DOWN (mirror PostUp with -D)
PostDown = iptables -t nat -D POSTROUTING ...

# Added automatically by add-client.sh:
[Peer]
# Name: alice
PublicKey    = <client public key>
PresharedKey = <shared symmetric key>
AllowedIPs   = 10.100.0.2/32, fd00:100::2/128

Essential Commands Cheat Sheet

# ── Service management ──────────────────────────────────────────────────
sudo systemctl status wg-quick@wg0      # Is VPN running?
sudo systemctl restart wg-quick@wg0     # Restart (reloads config)
sudo systemctl stop wg-quick@wg0        # Stop VPN
sudo systemctl start wg-quick@wg0       # Start VPN
sudo journalctl -u wg-quick@wg0 -f      # Follow live logs

# ── WireGuard status ─────────────────────────────────────────────────────
sudo wg show                            # All peers + stats
sudo wg show wg0 latest-handshakes      # When each peer last connected
sudo wg show wg0 transfer               # Bytes per peer

# ── Firewall ─────────────────────────────────────────────────────────────
sudo ufw status verbose                 # Show all rules
sudo iptables -t nat -L -n -v           # Check NAT/masquerade rules

# ── Diagnostics ──────────────────────────────────────────────────────────
curl https://api.ipify.org              # Your current public IP
ping 10.100.0.1                         # Ping VPN server from client
ip route show                           # Routing table
cat /etc/resolv.conf                    # Active DNS servers
Security reminder: After distributing a client config, delete it from the server: sudo shred -u -z /etc/wireguard/clients/alice/alice.conf. The client should be the only holder of their private key.
